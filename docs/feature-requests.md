# Feature Requests

Have an idea for improving **SF Flow Utility Toolkit**?

Feature requests are welcome and can help guide future development of the extension.

## Good feature requests usually explain

- the problem you are trying to solve
- why the current experience is not enough
- what you would like the extension to do
- where in Salesforce Flow Builder the feature would be useful
- whether the request would save time, reduce risk, or improve clarity

## Helpful examples

Useful feature requests often include details such as:

- a Flow Builder pain point
- the type of user who would benefit
- whether the feature belongs in the Flow list, Flow canvas, health tooling, or setup experience
- an example workflow the feature would improve

## Before submitting a request

Please consider:

- whether the idea fits the scope of the extension
- whether it overlaps with existing functionality
- whether the request is a new feature, an enhancement, or a usability improvement

## Submit a feature request

Use GitHub to suggest new features or enhancements for the project.

## What happens next

Feature requests may be discussed, refined, prioritised, or moved into tracked work items depending on scope and release plans.

Some requests may be small usability improvements, while others may become larger roadmap items.
