# Report a Bug

If you have found an issue in **SF Flow Utility Toolkit**, this page explains the best way to report it.

## Before submitting a bug

Before raising a bug report, please check the following:

- confirm that the issue can be reproduced
- check whether the extension was recently updated
- confirm which Salesforce page or Flow you were working in
- note which browser you are using
- check whether the issue affects one Flow or multiple Flows

## What to include

A useful bug report should include:

- a short summary of the problem
- the expected behaviour
- the actual behaviour
- steps to reproduce the issue
- browser name and version
- whether you were using Chrome or Edge
- any relevant screenshots
- any console errors, if available

## Salesforce-specific context

Because this extension works within Salesforce Flow Builder, it is especially helpful to include:

- the type of Flow involved
- whether the issue occurred on the Flow list, canvas, modal, or setup page
- whether the issue affects only one org or multiple orgs
- whether the issue is consistent or intermittent

## Sensitive information

Please do **not** include sensitive Salesforce data, credentials, session details, or confidential business information in public bug reports.

If a screenshot includes sensitive information, redact it before uploading.

## Submit a bug report

Use the GitHub issue tracker to submit bugs for this project.

## What happens next

Submitted bugs may be reviewed, clarified, reproduced, prioritised, and tracked through GitHub.

Not every bug will be fixed immediately, but clear reproduction steps make triage much easier.
